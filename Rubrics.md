| Batch | 24.2P |
| Module Lecturer | Niranga Dharmaratna |
| Assessment Setter | Niranga Dharmaratna |

### Learning Outcomes
1. Develop a secure, open-standards-based API to support server-client communication.  
2. Create modern web content involving asynchronous data retrieval, client-side DOM manipulation, standards adherence, and user-user interaction.  
3. Manage data persistence across both server and client web-based solutions.  
4. Design and implement an API and client based on given, non-trivial requirements using a range of appropriate developer tools.

## Assessment Rubric

### Criteria & Weighting
| Criterion | Weight | Level 1 | Level 2 | Level 3 | Level 4 | Level 5 |
|-----------|--------|---------|---------|---------|---------|---------|
| **API Design** | 20% | Attempt | Simple functional API | REST principles, basic auth | REST-compliant, filtering/sorting | Fully REST-compliant, conditional requests |
| **Solution Architecture** | 15% | Basic/ad-hoc | Some structure | Well-documented | Optimized, scalable | Robust, secure, monitored |
| **Implementation Code** | 15% | Attempt | Basic programming | Working modular code | Modular + exception handling | Full constructs, clear structure |
| **Version Control (GitHub)** | 10% | No access | Access but no commits | Some commits | Regular commits | Branching/merging, extended history |
| **Functionality** | 10% | None | Limited | Partial | Significant | Full coverage, tested |
| **Deployment** | 15% | None | Limited attempt | Deployed but inconsistent | Deployed with optimization | Fully deployed, best practices |
| **Report Quality** | 15% | None | Incomplete | Basic coverage | Detailed, organized | Comprehensive, well-structured |
